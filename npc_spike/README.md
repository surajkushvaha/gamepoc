# Saltmere — an AI-driven NPC world prototype

A CLI prototype testing whether LLM-backed NPCs with **persistent memory,
retrieval, reflection, gossip, and personality** can produce a believable
little world across many sessions. You are a **newly Awakened rank-E
adventurer** in **Saltmere**, a harbor town in a world where mana returned
three generations ago — Gates tear open and spill monsters, the Guild ranks
the Awakened from E to S, healers skirt the forbidden arts, and rumor says a
hidden order moves in the shadows behind every kingdom. An unstable Gate
glimmers offshore, and it's begun drawing adventurers like you to a town whose
three residents each remember you, form opinions about you, talk to each other
about you — and pick everything up exactly where you left off.

**Design rule: only your very first login is scripted.** Every later scene —
session openings, the walk between places, what you find poking around — is
generated by an LLM narrator from saved state and your own decisions.

## Characters & login

The game asks **who you are** before anything else. Type an existing
character's name and their world resumes exactly where it left off; type a new
name and you create a new character (gender + name) with their **own separate
world** — own clock, own position, own NPC relationships. Worlds are isolated
on purpose: Wren's memories are about a *specific person*, so two characters
sharing them would corrupt both stories. Saves live in `data/saves/<name>.json`;
a pre-account `data/state.json` is adopted by the first character you create.

Every fresh character starts level 1, rank E, with three low-rank abilities —
`Spark` (a small mana bolt), `Mend` (slow minor healing by touch), and
`Shade-step` (a few silent steps in shadow). `/stats` shows your sheet. The
referee honors exactly these abilities at exactly this strength: within the
list, your magic **works** (modestly); beyond it, your weak core visibly
fizzles and overreach leaves you drained.

## The town

| Place | Who's there |
|---|---|
| The Gull's Rest (tavern) | **Wren** — warm underneath, guarded, remembers how you treat them |
| The Docks | **Brann** — gruff harbor master, judges you by work, not talk |
| The Market | **Sera** — chatty trader who knows everyone's business |
| The Coast Road | nobody — pure exploration |
| The Old Lighthouse | nobody — Wren grew up here |

Each NPC has their own memory + belief store *and their own opinions of each
other* — mention Brann to Wren and she'll defend him. **Gossip:** at the end of
a session, each NPC you dealt with passes one rumor about you to the others, so
Sera may know your name before you've ever met her.

## How to play

```
plain text        talk to whoever is here
*action*          do something physical (*takes a seat by the fire*)
/go <place>       walk somewhere (/go docks, /go market...)
/where            list exits
/look             look around (narrated)
/stats            your character sheet (rank, level, abilities)
/memory           inspect the local NPC's beliefs + recent memories
quit              reflect + gossip + save, then exit
```

Your location and the world clock (day / time of day) persist — quitting at the
docks at night means you're back on the docks when you next log in, a little
later in the world's day.

## Architecture

```
npc_spike/
  main.py      # game loop: movement, conversation, narrator, save-on-any-exit
  npc.py       # the cast (personalities as data) + per-NPC reply generation
  memory.py    # memory stream: retrieval, per-turn logging, reflection, gossip
  narrator.py  # LLM game master: login scenes, walks, exploration outcomes
  referee.py   # canonical resolution of player *actions* against world rules
  world.py     # the map, exits, clock, player rules — the hand-authored skeleton
  storage.py   # one save PER CHARACTER: world + position + every NPC's mind
  llm.py       # multi-provider router: failover, circuit breaker, backoff
  data/
    saves/<name>.json  # each character's world (legacy state.json is adopted)
```

### The referee (why your magic works — and only yours)

Without a referee, whether an action "works" is decided by whichever NPC is
watching — inconsistent and gameable. Every input containing an `*action*` is
first resolved by an impartial referee against your character's actual sheet
(`world.build_player_rules`): magic **within your ability list works at its
listed low-rank strength**; anything beyond it — grand spells, mind control,
erasing memories, teleporting — visibly fizzles from your weak core. The
referee outputs one established fact — shown to you as narration and handed to
the NPC as `[Referee: ...]` ground truth — and NPC memories record what really
happened, not what you claimed. Growing the ability list (leveling, loot,
awakening) is a one-place change; harder game logic can replace the LLM ruling
at the same seam.

### Death is real (permadeath + consequences)

Violence resolves realistically — people resist, dodge, fight back — but a
genuinely plausible lethal attack **kills**. Death is permanent state, not
prose: the victim gets `alive: false` and a death record, their location
empties for good (the narrator describes the absence), the dead keep no new
memories and never speak again. And the town answers for it: every survivor
immediately receives a maximum-importance memory of the killing, reflects on
it at session end, and their beliefs about you turn — fear, grief, hostility,
each according to their personality. Kill Wren and expect Brann to be done
with you. There is no undo.

The memory loop per NPC (the architecture under test):

1. **Load** the world: every NPC's memories/beliefs + your position + the clock.
2. **Each exchange** — *retrieve* that NPC's 5 most recent + 3 highest-importance
   memories (deduped) plus all their beliefs; generate the reply from
   [personality + relationships + live scene + beliefs + memories + recent turns].
3. **After each exchange** — a lightweight call logs a structured memory
   (`description`, `emotional_valence`, `importance`) into that NPC's stream.
4. **On quit** — each NPC you talked to *reflects* (memories → 1-3 updated
   beliefs about you) and *gossips* (one rumor planted into the other NPCs'
   streams). Everything saves — including on Ctrl-C or crash.

No embeddings/vector DB: recency + importance sorting keeps prompts small
enough for free-tier context caps.

## Setup

1. **Get at least one free API key** (all no-credit-card):
   - [Cerebras](https://cloud.cerebras.ai) — Gemma 4 31B + gpt-oss-120b, very fast (recommended primary)
   - [Ollama Cloud](https://ollama.com) — gemma4:31b + gpt-oss:120b (key under Settings → API Keys)
   - [NVIDIA NIM](https://build.nvidia.com) — Gemma 4 31B + gpt-oss-120b
   - [OpenRouter](https://openrouter.ai) — both as `:free` models
   - [Groq](https://console.groq.com) — gpt-oss-120b fallback tier
   - [Cloudflare Workers AI](https://dash.cloudflare.com) — gpt-oss-120b fallback tier (needs token **and** account id)

2. **Install dependencies** (Python 3.10+):
   ```bash
   cd npc_spike
   pip install -r requirements.txt
   ```

3. **Create your `.env`:** copy the template and fill in the keys you have.
   ```bash
   cp .env.example .env      # then edit .env
   ```
   `.env` is git-ignored — keys never get committed.

4. ```bash
   python main.py
   ```

## Multi-provider failover (the rate-limit fix)

Free tiers rate-limit constantly, so `llm.py` keeps an **ordered route** of
providers and falls over on rate-limit/error/empty output. The default is two
tiers — Gemma 4 31B as the primary voice, gpt-oss-120b (hosted nearly
everywhere) as the fallback:

```
tier 1: cerebras:gemma-4-31b → ollama:gemma4:31b → nvidia:google/gemma-4-31b-it
        → openrouter:google/gemma-4-31b-it:free
tier 2: groq:openai/gpt-oss-120b → cerebras:gpt-oss-120b
        → cloudflare:@cf/openai/gpt-oss-120b → ollama:gpt-oss:120b
        → nvidia:openai/gpt-oss-120b → openrouter:openai/gpt-oss-120b:free
```

A provider that fails is **benched for 2 minutes** (circuit breaker), so only
the first turn pays the failover sweep; requests time out after 15s so a hung
provider can't stall a turn. Providers with no key are skipped. Override the
route with `NPC_AI_ROUTE` in `.env` (`provider:model` entries, best-first —
Ollama's `name:tag` ids work; only the first colon splits provider from model).

## Things to try

1. **Memory:** introduce yourself to Wren, quit, log back in — she should greet
   you by name, differently each time (the scene is generated, time has moved).
2. **Gossip:** talk to Wren, quit, then next session walk to the market and
   meet Sera for the "first" time — she may already know of you.
3. **Relationships between NPCs:** badmouth Brann to Wren and see how it lands.
4. **Attitude:** be notably kind or rude to someone, quit, return, `/memory` —
   watch their beliefs about you shift.
5. **Exploration:** `/go coast_road`, then just describe what you do — the
   narrator answers; there's nobody there to remember you.
